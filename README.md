# 날씨 앱

현재 위치 또는 검색한 장소의 날씨 정보를 확인하고, 즐겨찾기로 관리할 수 있는 웹 애플리케이션입니다.

## 프로젝트 실행 방법

### 환경 변수 설정

프로젝트 루트에 `.env` 파일을 생성하고 다음 환경 변수를 설정합니다:

```env
VITE_OPENWEATHER_API_KEY=your_openweather_api_key
VITE_KAKAO_API_KEY=your_kakao_api_key
```

### 설치 및 실행

```bash
설치 - npm install
실행 - npm run dev
```

## 구현한 기능

### 1. 현재 위치 날씨 확인

- 브라우저 Geolocation API를 활용하여 사용자의 현재 위치를 자동으로 감지합니다.
- 위치 권한을 허용하면 해당 좌표의 현재 날씨와 5일간의 예보를 확인할 수 있습니다.
- 위치 권한이 거부된 경우 안내 메시지를 표시합니다.

### 2. 위치 검색

- 대한민국 행정구역(시/군/구) 기반 위치 검색 기능을 제공합니다.
- 검색어 입력 시 디바운싱(300ms)을 적용하여 불필요한 API 호출을 방지합니다.
- 검색 결과 선택 시 해당 위치의 날씨 정보를 즉시 표시합니다.

### 3. 즐겨찾기 관리

- 검색한 장소를 즐겨찾기에 추가할 수 있습니다.
- 즐겨찾기 목록에서 각 위치의 현재 날씨를 한눈에 확인할 수 있습니다.
- 즐겨찾기 이름 수정 및 삭제 기능을 제공합니다.
- 동일한 위치의 중복 등록을 방지합니다.

### 4. 날씨 상세 페이지

- 즐겨찾기 카드 선택 시 해당 위치의 상세 날씨 페이지로 이동합니다.
- 현재 날씨와 함께 5일간의 3시간 단위 예보를 확인할 수 있습니다.
- 시간대별 기온, 날씨 상태, 강수 확률 등 상세 정보를 제공합니다.

### 5. 반응형 디자인

- 모바일과 데스크톱 환경 모두에서 최적화된 UI를 제공합니다.
- 모바일에서는 Drawer 형태의 사이드바를 통해 즐겨찾기 목록에 접근합니다.
- 데스크톱에서는 고정된 사이드바로 즐겨찾기 목록을 항상 표시합니다.

## 기술적 의사결정 및 이유

### React Query를 통한 서버 상태 관리

**이유**:
- 자동 캐싱으로 동일한 API 요청의 중복 호출을 방지합니다.
- Suspense 통합으로 선언적인 로딩 상태 처리가 가능합니다.
- staleTime 설정으로 날씨 데이터의 적절한 갱신 주기를 관리합니다.

### Zustand를 통한 클라이언트 상태 관리

**이유**:
- 가볍고 직관적인 API로 전역 상태를 간편하게 관리할 수 있습니다.
- persist 미들웨어를 활용하여 즐겨찾기 데이터를 LocalStorage에 자동 동기화합니다.
- React Query와 역할을 분리하여 서버 상태와 클라이언트 상태를 명확히 구분합니다.

### 디바운싱 적용

**이유**: 검색 입력 시 매 키 입력마다 API를 호출하면 불필요한 네트워크 요청이 발생합니다. 300ms 디바운싱을 적용하여 사용자가 입력을 멈춘 후에만 검색 API를 호출하도록 최적화했습니다.

### Code Splitting (Lazy Loading)

**이유**: 상세 페이지를 `React.lazy`로 동적 임포트하여 초기 번들 크기를 줄이고, 해당 페이지 접근 시에만 코드를 로드하도록 했습니다.

### Error Boundary 적용

**이유**: 컴포넌트 트리 내에서 발생하는 에러를 격리하고, 사용자 친화적인 에러 화면을 제공합니다. 전체 앱이 크래시되지 않고 해당 영역만 에러 처리됩니다.
